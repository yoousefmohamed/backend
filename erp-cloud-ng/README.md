# ☁️ ERP Cloud Next Generation

نظام ERP سحابي متكامل — يعمل أونلاين بالكامل، يدعم الكمبيوتر والهاتف والتابلت.

## البنية التقنية

| الطبقة | التقنية |
|---|---|
| الواجهة الأمامية | Next.js 14 + React 18 |
| الخادم | NestJS + Node.js |
| قاعدة البيانات | PostgreSQL 16 |
| التحديثات اللحظية | Socket.io (WebSocket) |
| المصادقة | JWT (access + refresh tokens) |

## التشغيل السريع (Docker — يُنصح به)

```bash
docker compose up -d --build
```

بعد التشغيل:
- الواجهة: http://localhost:3000
- الـ API: http://localhost:3001/api/v1
- وثائق Swagger: http://localhost:3001/api/docs

قاعدة البيانات تُهيأ تلقائيًا من `database/schema.sql` و `database/seed.sql` عند أول تشغيل، وتتضمن شركة تجريبية وحسابات الأدوار الستة.

## التشغيل اليدوي (بدون Docker)

### 1. قاعدة البيانات

```bash
createdb erp_cloud_ng
psql erp_cloud_ng -f database/schema.sql
psql erp_cloud_ng -f database/seed.sql
```

### 2. الخادم (Backend)

```bash
cd backend
cp .env.example .env   # عدّل بيانات الاتصال بقاعدة البيانات إذا لزم
npm install
npm run start:dev
```

### 3. الواجهة (Frontend)

```bash
cd frontend
cp .env.local.example .env.local
npm install
npm run dev
```

## بيانات الدخول التجريبية

بعد تشغيل `seed.sql`:
- **معرّف الشركة:** `11111111-1111-1111-1111-111111111111`
- **اسم المستخدم:** `admin`
- **كلمة المرور:** يجب ضبطها أولًا — السجل في `seed.sql` يحتوي على bcrypt hash مؤقت، استخدم نقطة `/api/v1/users` لإنشاء مستخدم admin جديد بكلمة مرور حقيقية، أو حدّث `password_hash` مباشرة في قاعدة البيانات.

## الوحدات المُنفّذة

- **المصادقة والصلاحيات:** JWT + نظام صلاحيات متعدد المستويات (وحدة × صلاحية) قابل للتخصيص بالكامل لكل دور.
- **تعدد الشركات والفروع:** بنية multi-tenant كاملة.
- **المنتجات والمخازن:** باركود، تصنيفات، حد أدنى للمخزون، تحويل بين المخازن، حركة مخزون مدققة (stock_movements).
- **نقطة البيع (POS):** فواتير نقدي/آجل/تقسيط، خصم المخزون التلقائي، تحديث الخزنة وحساب العميل تلقائيًا، بث لحظي عبر WebSocket.
- **العملاء والموردون:** كشف حساب، حدود ائتمانية، أرصدة.
- **المشتريات:** فواتير شراء تضيف للمخزون وتُحدّث رصيد المورد تلقائيًا.
- **الإنتاج:** قائمة مكونات (BOM) لكل منتج، أوامر إنتاج تخصم الخامات وتضيف المنتج النهائي تلقائيًا مع حساب التكلفة.
- **الحسابات:** دفتر الأستاذ (journal entries)، ميزان المراجعة، قائمة الدخل، الميزانية العمومية.
- **الخزنة والبنوك:** إيداع/صرف، تتبع الرصيد لحظيًا.
- **التقارير:** مبيعات، مشتريات، أرباح وخسائر، مخزون، كشف عملاء، إنتاج — قابلة للاستهلاك وتصديرها لاحقًا PDF/Excel من الواجهة.
- **App Builder:** إنشاء شاشات وحقول مخصصة (نص، رقم، تاريخ، قائمة، مرفق، ربط، اختيار، حقل محسوب) بدون كود، مع جداول بيانات JSONB مرنة.
- **الإشعارات وسجل العمليات:** تنبيهات لحظية + سجل تدقيق كامل لكل عملية.

## خارطة الطريق (لم يُنفذ بعد)

- تطبيق Flutter للأندرويد/آيفون (يمكن استهلاك الـ API الحالي مباشرة).
- تكامل واتساب الفعلي لإرسال الفواتير (الحقول جاهزة في قاعدة البيانات: `whatsapp_phone`).
- نماذج الذكاء الاصطناعي للتنبؤ بالمبيعات واقتراح إعادة الشراء (التصميم التفاعلي جاهز، يحتاج خدمة تنبؤ منفصلة).
- طباعة حرارية فعلية (ESC/POS) — حاليًا تصميم HTML قابل للطباعة فقط.
- PWA manifest + service worker للعمل دون اتصال.

## هيكل المشروع

```

├── backend/          # NestJS API
│   └── src/modules/  # كل وحدة في مجلد مستقل (CRUD + منطق أعمال)
├── frontend/          # Next.js App Router
│   └── src/app/       # صفحة لكل مسار
├── database/
│   ├── schema.sql      # كل الجداول والفهارس
```
